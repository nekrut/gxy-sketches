# Immunopeptidomics DDA peptide identification and quantification

## When to use this sketch
- You have DDA mass spectrometry runs of MHC/HLA-eluted peptides (class I or class II) and need an FDR-controlled peptide list.
- Inputs are Thermo `.raw`, `.mzML`/`.mzML.gz`, or Bruker timsTOF `.d` (optionally `.d.tar.gz`/`.d.zip`) plus a protein FASTA (e.g. UniProt/SwissProt human).
- Peptides were generated by MHC processing (unspecific cleavage) rather than a protease like trypsin, so typical lengths are 8-12 aa (class I) or 8-25 aa (class II).
- You want label-free quantification, global FDR across samples, spectrum-library generation for downstream DIA, ion annotation, or consensus-epitope (epicore) collapsing of length variants.
- You are reprocessing a public immunopeptidomics dataset and need a reproducible, container-based pipeline.

## Do not use when
- Your data are DIA rather than DDA — this pipeline searches DDA spectra; use a DIA-specific workflow and, if needed, feed it a spectrum library generated here via `--generate_speclib`.
- You are doing standard tryptic shotgun proteomics or PTM discovery on whole-cell lysates — use a general proteomics DDA pipeline (e.g. nf-core/quantms) with enzyme `Trypsin` rather than `unspecific cleavage`.
- You need de novo sequencing without a database, or cross-linking MS, or TMT/iTRAQ isobaric quantification workflows.
- Your input is genomics/transcriptomics (FASTQ, BAM) rather than mass spectrometry spectra.

## Analysis outline
1. **Spectra preparation** — convert/centroid raw, mzML, or Bruker `.d` inputs via the `PREPARE_SPECTRA` subworkflow (OpenMS `PeakPickerHiRes` when `--run_centroidisation`).
2. **Decoy database** — generate a reversed target-decoy FASTA with OpenMS `DecoyDatabase` (skip with `--skip_decoy_generation` if decoys already prefixed `DECOY_`).
3. **Database search** — run Comet via OpenMS `CometAdapter` with unspecific cleavage and immunopeptide-appropriate mass/charge ranges.
4. **Peptide indexing** — refresh protein references and annotate target/decoy with `PeptideIndexer`.
5. **ID merging** — merge runs sharing the same `Sample`/`Condition` with `IDMerger`.
6. **Rescoring** — `MS²Rescore` adds DeepLC retention-time and MS²PIP fragment-intensity features (optionally Ionmob/IM2Deep CCS for timsTOF), then `PSMFeatureExtractor` and `PercolatorAdapter` (or Mokapot) rescore PSMs; `IDFilter` applies the FDR threshold.
7. **Export** — write FDR-filtered peptides to TSV via `TextExporter`; when `--quantify`, also emit `mzTab`.
8. **Optional quantification** (`--quantify`) — retention-time align runs (`MapAlignerIdentification` + `MapRTTransformer`), then `FeatureFinderIdentification` → `FeatureLinkerUnlabeledKD` → `IDConflictResolver` for label-free peak-area quantification with match-between-runs.
9. **Optional spectrum library** (`--generate_speclib`) — build per-sample (or global when combined with `--global_fdr`) DIA-ready libraries with EasyPQP.
10. **Optional ion annotation and epicore** (`--annotate_ions`, `--epicore`) — annotate MS2 b/y ions per PSM and collapse overlapping length variants into consensus core epitopes.

## Key parameters
- `--input` (tsv with columns `ID`, `Sample`, `Condition`, `ReplicateFileName`, optional `Fasta`) and `--fasta` (protein database).
- `--enzyme` default `unspecific cleavage` (correct for HLA peptides; do not switch to `Trypsin` unless you actually digested).
- `--instrument` `high_res` (Orbitrap/timsTOF) or `low_res` (ion trap). Controls Comet fragment-ion modeling.
- `--activation_method` `HCD` for Orbitrap, `CID` for timsTOF/ion-trap, or `ALL` to auto-infer.
- `--precursor_mass_tolerance` 5 ppm (Orbitrap high-res) / 20 ppm (timsTOF); `--precursor_error_units ppm`.
- `--fragment_mass_tolerance` 0.02 for high-res (schema default 0.01 works for Orbitrap), ~0.50025 with `--fragment_bin_offset 0.4` for low-res ion traps.
- `--prec_charge` `2:3` (class I Orbitrap), `2:5` (class II), `1:4` or `1:5` for timsTOF.
- `--digest_mass_range` `800:2500` for class I, `800:5000` for class II.
- `--peptide_min_length` / `--peptide_max_length` `8`/`12` for class I, extend upper bound (~25) for class II.
- `--fixed_mods` typically empty for HLA peptides; `--variable_mods` default `Oxidation (M)`; use full UNIMOD names.
- `--fdr_level` `peptide_level_fdrs` (default) and `--fdr_threshold 0.01`.
- `--rescoring_engine` `percolator` (default) or `mokapot`; `--feature_generators deeplc,ms2pip` (add `ionmob`/`im2deep` for timsTOF CCS).
- `--ms2pip_model` `Immuno-HCD` for HCD immunopeptidomics; pick `timsTOF`/`timsTOF2023` for Bruker data, `CID` for ion-trap.
- Feature toggles: `--quantify`, `--global_fdr` (percolator only), `--generate_speclib`, `--annotate_ions`, `--epicore` (+ `--epicore_min_overlap`, `--epicore_max_step_size`, `--epicore_min_epi_length`).

## Test data
The `test` profile runs on a small HepG2 immunopeptidomics samplesheet (`mhcquant/testdata/HepG2_sample_sheet.tsv`) searched against a trimmed UniProt human proteome (`UP000005640_9606.fasta`), both pulled from the nf-core test-datasets repository. Defaults exercise the identification path only (Comet → MS²Rescore → Percolator → IDFilter) and should produce per-sample/condition FDR-filtered peptide TSVs plus a MultiQC report under `--outdir`. The `test_full` profile uses `sample_sheet_full.tsv` with `--quantify --generate_speclib --annotate_ions --epicore` and both Oxidation (M) and Carbamidomethyl (C) as fixed and variable mods, additionally emitting an `mzTab` with peak areas, per-sample `*_speclib.tsv` libraries, ion-annotation TSVs under `intermediate_results/ion_annotations/`, and epicore consensus-epitope columns in the final peptide TSV.
