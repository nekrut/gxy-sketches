# Per-class motif mining

Sketches classified into fine-grained workflow classes (sub-domains), then frequent k-paths mined within each class. Class-internal motifs are the actually-reusable modules — generic cross-domain motifs (qc, alignment) are deliberately suppressed by the per-class scope.

Generated by `scripts/classify_and_mine.py` (which reuses extraction from `scripts/extract_dag_motifs.py`). Raw output in `scripts/class_motifs.json`.

The clean wins are **vc-haploid-microbial**, **vc-viral-amplicon**, **vc-viral-shotgun**, **epi-chipseq**, **epi-cutandrun**, **epi-hic**, **phylo**: in each, the motif at full or near-full class support recovers the textbook pipeline. These are the classes ready for module extraction.

## What this pass exposed that the cross-corpus pass missed

- **`vc-amplicon-viral` is genuinely distinct from `vc-viral-shotgun`.** The cross-corpus pass conflated them; `ivar` (vc_amplicon_viral) only appears in amplicon sketches and is *additive* to `lofreq`, not a substitute. Module catalogue should split M16 into M16a (shotgun) and M16b (amplicon).
- **HyPhy chain.** Six tools called in fixed order across multiple viral phylogenetics sketches — invisible at the corpus level because phylo only has 7 sketches, but at full support within phylo. Strong candidate for a `phylo-codon-selection` module.
- **Hi-C is its own thing.** `hicup → hicup2juicer → cooler` is a self-contained pipeline that doesn't share with ChIP/ATAC despite living in the same epigenomics domain.
- **bowtie2 is the ChIP/ATAC default**, bwa-mem* is the variant-calling default. Per-class mining makes this explicit (see `epi-chipseq` motif vs. `vc-*` motifs).

## Limitations

- **Coverage.** Only 175/258 sketches have parseable analysis outlines (bold-marked tools in numbered steps). Several classes (e.g. `prot-dda`, `annot-eukaryotic`, `meta-tax-profiling`) are under-represented in the parsed set even though they exist as classes. An LLM extraction pass would close this.
- **Linearisation.** Steps that run in parallel are listed sequentially in prose, which inflates linear-chain support and produces some semi-spurious longer motifs (e.g. the `rna-bulk-de` chain mixing `cufflinks` and `count`). Real DAG edges (LLM-extracted `depends_on`) would fix this.
- **Class size.** Classes with <3 parsed sketches don't yield motifs; mining there is a no-op. Either lump them with `*-other` or boost via LLM extraction.
- **Class boundaries.** Hand-written rules; some sketches will land in `*-other` that arguably belong elsewhere. Review the classification table below before relying on a specific motif.
